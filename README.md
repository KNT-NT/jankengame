# jankengame

## 概要
オブジェクト指向プログラミングの学習を目的とした、コンソール上で動作するじゃんけんゲームです。

### 開発の背景や要素
「単に動くコード」ではなく、「変更に強く、役割が明確なコード」を目指しました。  
じゃんけんの要素を「主体(手)」「判定(審判)」「進行(メイン)」に分解し、オブジェクト間の責務を意識して設計しました。

### こだわった点
1. `Judge`クラスでは大量の`if-else`文を避けるために、`HashMap`を用いて勝敗パターンを定義しました。  
2. 「あいこ」の際のループ処理において、再帰呼び出しを避けるために`do-while`文による反復処理にしました。  
3. 入出力（`Main`）、データ管理（`Hand`）、ロジック（`Judge`）をそれぞれのクラスに切り出し、クラス間の役割を明確に分けました。
4. (新規)`getter/setter`を導入し、カプセル化を実装しました。

## 構成・設計
### クラス構成
本アプリは以下の3つのクラスで構成されています。  

| クラス名 | 役割 |
| :--- | :--- |
| **Main** | ゲーム進行の制御。ユーザー入力の受付、各クラスのインスタンス化、結果の出力。 |
| **Hand** | 手の状態管理。ランダム生成、および数値から文字列への変換をします。 |
| **Judge** | 勝敗判定の実行。Mapに定義された勝敗パターンに基づき結果を出力します。 |
